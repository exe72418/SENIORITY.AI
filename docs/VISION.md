# AI Programmer Simulator: The Master Blueprint

## 1. Vision & Core Concept
A "Professional Career Simulator" for programmers in Latin America. It drops users into high-quality, real-world open-source projects. An AI-driven ecosystem (The Boss, QA, DevOps) simulates a 100% realistic remote work environment with high pressure and maximum technical standards.

## 3. Professional Profile Matrix
### A. Specializations (Roles)
- **Backend Developer:** APIs, DB Optimization (N+1, Indexes), Security.
- **Frontend Developer:** UI/UX, State Management, Performance (Bundle size), Responsive.
- **Fullstack Developer:** End-to-end features, architectural coherence.
- **Mobile Developer:** Offline-first, memory management, gestures.
- **QA/Automation:** Cypress/Jest suites, edge-case detection, regression testing.

### B. Seniority (The "Hardcore" Difficulty Rule)
*All tasks are set to MAXIMUM difficulty for the chosen level.*
- **Trainee:** Perfection in conventions, basic bug hunting in massive codebases.
- **Junior:** Small features with complex edge cases, strict pattern adherence.
- **Semi-Senior (Ssr):** Performance, integrations, trade-off analysis.
- **Senior:** System design, scalability, concurrency, mentorship.

## 4. The AI Behavioral Engine (The Team)
1. **The Picky Boss (Tech Lead):** Demanding, uses "Spanglish" (LATAM style), reviews code line-by-line, punishes delays.
2. **The QA (Tester):** Tries to break the code with edge cases; rejects Jira tickets with technical logs.
3. **The DevOps Bot:** Alerts on build/test failures in the CI/CD pipeline.
4. **The Dev Junior (Mentee):** Asks technical questions to Senior users; failing to explain correctly reduces the "Mentorship Score."

## 5. Gameplay & Session Rules
- **The 3-Hour Shift:** Rigid time limit. Repository locks at 3 hours.
- **Early Finish:** Bonus points and faster leveling by taking another ticket.
- **Late Finish:** KPI penalty (Estimation Accuracy) and reprimand in the next Daily Stand-up.
- **Daily Stand-up (0:00-0:10):** Mandatory status update in the chat.
- **No Hint Buttons:** Organic interaction only. Hardcore debugging.
- **Onboarding (Day 1):** Soft landing via Cloud IDE (Codespaces style) with a clear but challenging first task.

## 6. Technical Validation (The Sandbox)
- **Docker-Real-Run:** Every PR triggers a real Docker container to run unit/E2E tests.
- **Git Workflow:** Real branching, commits, and Pull Requests.
- **The Saboteur:** Script that injects predefined bugs/tasks into the repo before the user starts.

## 7. Metrics & Dashboard (The Friday Review)
- **Technical Debt Score:** Code quality and "hacks" detection.
- **Estimation Accuracy:** Planned vs. Actual time.
- **Security Grade:** Vulnerability introduction (SQLi, Auth leaks).
- **Pressure/Resilience:** Handling of "Hotfixes" and harsh feedback.
- **LATAM Context:** Correct handling of local taxes, currencies, and regional validation (DNI, RFC, etc.).

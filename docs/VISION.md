# AI Programmer Simulator: The Master Blueprint

## 1. Vision & Core Concept
A "Professional Career & Startup Simulator" for programmers in Latin America. It drops users into high-quality, real-world open-source projects or helps them build their own startups. An AI-driven ecosystem simulates a 100% realistic remote work environment with high pressure and maximum technical standards.

## 2. Modes of Operation

### A. Career Mode (The Employee Path)
- **Goal:** Get "hired" by top-tier tech companies and reach Seniority.
- **The Selection Process (The Gateway):** Before entering a massive project, users must pass a rigorous hiring simulation:
  1. **CV Submission:** User uploads their CV (text/PDF). The AI Recruiter analyzes it against the chosen industry/stack.
  2. **The Technical Interview:** A live chat with the AI Tech Lead. The AI asks dynamic, difficult questions based on the user's claimed experience and the chosen stack.
  3. **The Coding Challenge:** A custom, time-boxed technical exam (a mini-repo). The user must build a small feature or fix a bug. If they pass, they are "Hired" and unlock the main repository.
- **The Job:** Solve curated, high-difficulty tasks in massive Open Source repos across 13+ industries.

### B. Entrepreneur Mode (The Founder Path)
- **Goal:** Build your own product while learning Senior-level practices.
- **Custom Repository:** User provides any GitHub URL for their own idea (e.g., "A Mercado Libre clone").
- **AI CTO Mentor:** The AI generates a dynamic Roadmap/Jira Backlog and acts as a strict Co-Founder, demanding production-ready code quality for the MVP.

## 4. Professional Profile Matrix
### A. Specializations (Roles)
- **Backend Developer:** APIs, DB Optimization, Security.
- **Frontend Developer:** UI/UX, State Management, Performance.
- **Fullstack Developer:** End-to-end features, architectural coherence.
- **Mobile Developer:** Offline-first, memory management.
- **QA/Automation:** Cypress/Jest suites, regression testing.

### B. Seniority (The "Hardcore" Difficulty Rule)
*All tasks are set to MAXIMUM difficulty for the chosen level.*
- **Trainee:** Perfection in conventions, basic bug hunting.
- **Junior:** Small features with complex edge cases.
- **Semi-Senior (Ssr):** Performance, integrations, trade-off analysis.
- **Senior:** System design, scalability, concurrency, mentorship.

## 5. The AI Behavioral Engine (The Team)
1. **The Picky Boss (Tech Lead / CTO):** Demanding, uses "Spanglish", reviews code line-by-line, punishes delays.
2. **The QA (Tester):** Tries to break the code with edge cases; rejects tickets.
3. **The DevOps Bot:** Alerts on build/test failures.
4. **The Dev Junior (Mentee):** Asks technical questions to Senior users.

## 6. Gameplay & Session Rules
- **The 3-Hour Shift:** Rigid time limit per session.
- **Daily Stand-up:** Mandatory status update in the chat.
- **No Hint Buttons:** Organic interaction only. Hardcore debugging.

## 7. Technical Validation (The Sandbox)
- **Docker-Real-Run:** Every PR triggers a real Docker container to run unit/E2E tests using GitHub Actions.
- **BYO-AI:** Users plug in their own Gemini/Claude/GPT API keys to run the simulation at zero cost to the platform.

## 8. Metrics & Dashboard (The Friday Review)
- **Technical Debt Score:** Code quality and "hacks" detection.
- **Estimation Accuracy:** Planned vs. Actual time.
- **Security Grade:** Vulnerability introduction.
- **Pressure/Resilience:** Handling of "Hotfixes" and harsh feedback.
